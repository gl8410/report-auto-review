Now let's go to module 5, called the **"Historical Report Intelligent Analysis & Rule Extraction Engine"**.
The purpose of this module is to perform "reverse engineering" on engineering reports by comparing the "Draft Version" (Before) with the "Approved Version" (After). The system must utilize AI to infer the "Expert Review Opinions" that caused the changes, allow human validation, and store these opinions as standardized rules in a specific "ruleset" for future automated checks.

# Workflow Description (User Interaction)
1.  **File Upload & Configuration**:
    * User selects the existing rule group from rule management module
    * User uploads two PDF files:
        * File A: `Original Draft` (Before,"原始稿件")
        * File B: `Approved Final Version` (After,"修改后稿件")
        !! 2 types of docs can be multiple files
2.  **Intelligent Analysis (AI Processing)**:
    * Extract text content from both PDF or docx files.
    * Use an LLM to perform a semantic comparison (not just a text diff, but a semantic evolution analysis).
    * **Core Task**: Based on the differences, the LLM must *infer* the expert opinion that dictated the change.
        * *Example*: If the Approved Version adds a calculation for "Pipeline Occupation Area" that was missing in the Draft, the AI should infer the opinion: "Please supplement the calculation process and basis for the area of the river channel occupied by the pipeline."
3.  **Human-in-the-Loop Review**:
    * Frontend displays a workspace: Left side shows "AI Inferred Opinions", Right side shows "Text Diff Evidence" (Before vs. After snippets).
    * User allows editing, correcting, or deleting the inferred opinions.
    * User clicks "Confirm & Add to Rule Base".
4.  **Rule Storage**:
    * Save the validated opinions into the selected rule group as structured rules.(similar as in module 4 for reviewing comments)
